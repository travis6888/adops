AD OPTIMIZER

This is an internal tool that will help increase productivity by decreasing the amount spent on sorting ad campaigns and individual ads

    Some important notes:
  • This system only works with Excel workbooks, specifically the .xls format. The .xlsx format may work but is not guaranteed.
  • Files over 15mb's will break the system it seems, it is best to copy the clean ad report to a new workbook.
  • Make sure the formulae went all the way through the sheet from top to bottom for things like SU/IMP
  • To help make the program run smoothly it is best to copy & paste special values to the sheet you wish to parse. The copy and paste will remove any formulae from the sheet.
  • Columns are represented as integers and column A starts at 0. Make sure to have the correct column location for each input, examples are A=0, J=9, etc.
  • You must label the sheet number too figure out which sheet to parse. The first sheet is 1, and continues normally from there.
  • Name the file anything you want, the file returned will only have one sheet and will be called adoptimization with todays date.
  • The page does not disappear and the download will happen automatically.
  • The input fields set floors to sort ads. SU's of 3 will make any ad sort ads with less than 3 differently than those with 3 or more.
  • There is multi stage sorting happening.
  • An example of a performing ad may have a click thru rate(CTR) >= 5%, SU/IMP >= 1%, Clicks >= 30 , Impressions >= 50, SU's >= 5.
  • So it best to set the input fields as according to your KPI's.
  • The reason to set SU, Clicks, and Impression floors is to make sort based on statistical significance. Ads with low amounts of data give skewed results.
  • CTR and SU/IMP need to represented in decimal form, .01 for 1%, etc.
